注意：
1. 输入的mesh不要缩放，否则导致生成的prefab无法找到（由于被缩放，比如缩放为0.01）；
2. 动画需要是legacy ；
3. 当然这种方式要求Vertex的textrue fetch，要求GPU支持float类型的贴图，一般需要gles3.0以上级别的机器（当然也已经算比较普遍）；
4. gpu instance需要opengl es3.0；
5. 材质球的Enable instance记得要勾选上，这样才能合批。

使用说明：
1. 单击菜单3DArt->AnimMapBaker，打开工具窗口；
2. 将包含skinmesh，Animation组件的，mesh没有被缩放的物体（最好是有主贴图的，这样生成的材质球里也会填入主贴图），拖拽到窗口的第一个空格；
3. 在第二个空格中填入生成的资源相对于Asset的路径，比如路径为Asset/Res，则需要输入Res，这里设置的默认路径是“CapstonesRes/Game/Models/Scene/Flag”，可以通过拖拽文件夹，自动获取文件夹路径；
4. 第三个空格拖入事先做好的材质球（设置好了主贴图等除了跟动画相关的参数之外的所有参数），生成新的材质球的时候，就能根据这个材质球进行复制；
5. 保存的时候，会弹框“是否保存对xx材质球的修改”，点击“是”；
6. 保存路径自动设为prefab，这样就会自动生成所需要的prefab，存储节点位置的贴图和材质球，可以直接使用。   测试
